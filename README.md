# 廃人サバメンバーキャラ武器所持状況チェック

原神のサーバーメンバーのキャラクター・武器所持状況を一元管理するためのサイトです。  
複数端末・複数人から同じデータを参照・編集できます。

## 機能概要

- メンバーの追加・削除
- キャラクターごとの所持有無・凸数（未所持 / 無凸〜6凸）・レベル（1〜90）の管理
- 武器ごとの所持有無・精錬数（未所持 / 精錬1〜5）の管理
- 元素・武器種・凸数・精錬数でのフィルタリング
- 凸数順・精錬数順のソート
- Supabase を使った複数端末間でのリアルタイム共有保存

## セットアップ手順

### 1. Supabase プロジェクトを作成する

1. [supabase.com](https://supabase.com) でアカウントを作成し、新しいプロジェクトを作成します。
2. プロジェクトの **Settings → API** を開き、以下の値をメモしておきます：
   - **Project URL**（例: `https://xxxxxxxxxxx.supabase.co`）
   - **anon (public) key**（例: `******

### 2. データベーステーブルを作成する

1. Supabase ダッシュボードの **SQL Editor** を開きます。
2. このリポジトリの **`supabase-schema.sql`** の内容をそのまま貼り付けて実行します。
3. `members`・`member_character_status`・`member_weapon_status` の3テーブルが作成されます。

### 3. config.js を作成する

1. リポジトリ内の `config.example.js` をコピーして、同じディレクトリに `config.js` という名前で保存します。
2. `config.js` を開き、以下のように Supabase の接続情報を入力します：

```js
window.SUPABASE_URL = 'https://xxxxxxxxxxx.supabase.co';
window.SUPABASE_ANON_KEY = '******';
```

> **注意:** `config.js` は `.gitignore` に登録されているため、リポジトリに含まれません。接続情報を誤ってコミットしないようご注意ください。

### 4. サイトを開く

`index.html` をブラウザで開くか、GitHub Pages 等で公開すれば使用できます。

## データ仕様

### キャラクター（constellation 列）

| 値    | 意味   |
|-------|--------|
| NULL  | 未所持 |
| 0     | 無凸   |
| 1〜6  | N凸    |

### キャラクター（level 列）

| 値    | 意味                 |
|-------|----------------------|
| 1〜90 | 所持キャラクターのレベル |

### 武器（refinement 列）

| 値    | 意味   |
|-------|--------|
| NULL  | 未所持 |
| 1〜5  | 精錬N  |

---

## Supabase テーブル設計

| テーブル名          | 概要                         |
|---------------------|------------------------------|
| members             | メンバー名を管理             |
| member_character_status | メンバーごとのキャラ所持状況 |
| member_weapon_status    | メンバーごとの武器所持状況   |

RLS（Row Level Security）は全員読み書き可能（認証なし）に設定されています。

---

## 必要な環境変数（config.js）

| 変数名              | 説明                                     |
|---------------------|------------------------------------------|
| SUPABASE_URL        | Supabase プロジェクトの URL              |
| SUPABASE_ANON_KEY   | Supabase の匿名（公開）キー              |

## 今後調整しやすいポイント

- **キャラクター・武器データ**: `script.js` 冒頭の `characters` 配列と `allWeapons` オブジェクトを編集することで追加・修正できます。
- **表示するキャラクター・武器の絞り込み**: 現在は ★4/★5 武器のみ表示しています。`script.js` の `renderWeaponTab()` の `w.rarity < 4` 条件を変更することで調整できます。
- **アクセス制御**: 現在はログインなしで全員編集可能です。アクセス制限が必要な場合は Supabase の RLS ポリシーを変更してください。
- **元素順・武器種順**: `ELEMENT_ORDER` / `WEAPON_TYPE_ORDER` 定数を編集することで並び順を変更できます。
